# Normative Scope Protocol

## Purpose

This protocol defines the normative scope of `SAD-MORON-FRAMEWORK`.

## Boundary

This protocol governs the outer institutional and regulatory limits of the framework layer.

It does not define:
- implementation design
- operational procedures in full detail
- unrestricted data collection
- technical roadmaps beyond governance proportionality

## Responsibilities

This protocol is responsible for defining:
- the regulatory boundary relevant to the framework
- the SAD operational scope to which governance applies
- proportional infrastructure growth expectations
- explicit exclusions that protect the framework from overreach

## Exclusions

This protocol explicitly excludes:
- unrestricted data capture
- undefined institutional domains outside SAD scope
- technical expansion without governance need
- generic platform growth disconnected from educational administration

## Enforcement Rules

### Rule 1: Regulatory boundary

The framework is bounded by Provincia de Buenos Aires educational regulations as the primary normative context for institutional governance interpretation.

### Rule 2: SAD operational scope

The framework applies only to SAD Moron operational governance concerns and should not silently generalize itself into unrelated institutional domains.

### Rule 3: Proportional infrastructure growth

Infrastructure growth must remain proportional to defined governance needs and must not outrun the framework's normative clarity.

### Rule 4: No unrestricted data capture

The framework rejects unrestricted data capture as a governance model. Data relevance must remain bounded by declared source, expected event, period, and institutional purpose.

## Summary

Normative scope in SAD Moron is intentionally bounded by educational regulation, institutional purpose, and proportional governance growth.
