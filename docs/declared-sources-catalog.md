# Declared Sources Catalog

## Purpose

This document establishes the initial declared source catalog for `SAD-MORON-FRAMEWORK`.

## Status

This is an initial catalog, not a final catalog.

It exists to provide the minimum declared-source anchor required for bounded governance review before operational repositories are created.

## Catalog Entry Format

Each source entry records:
- source name
- institutional origin
- document or record class
- intended governance use
- temporal relevance
- known limitations
- boundary note

## Initial Catalog

### 1. SAD Moron Administrative Records

- Source name: SAD Moron administrative records
- Institutional origin: Secretaria de Asuntos Docentes de Moron
- Document or record class: internal administrative records, registries, process records, filing traces
- Intended governance use: evidence of institutional administrative actions, expected-event verification, bounded omission review
- Temporal relevance: active across current and historical administrative periods depending on record retention
- Known limitations: completeness may vary by process, medium, and historical migration state
- Boundary note: only records formally recognized by SAD Moron administrative practice should be treated as authoritative within this source class

### 2. POFA / POF Documents

- Source name: POFA / POF documents
- Institutional origin: institutional and provincial educational staffing and organizational processes
- Document or record class: staffing, functional allocation, and organizational framework documents
- Intended governance use: evidence for expected staffing-related events, structural administrative verification, and context for review
- Temporal relevance: tied to the administrative cycle, issuance period, and subsequent formal updates
- Known limitations: may require contextual interpretation across versions and administrative periods
- Boundary note: derivative copies or informal reproductions must not be treated as equivalent to formally recognized versions without declared basis

### 3. ABC / Provincial Administrative Platform

- Source name: ABC / provincial administrative platform
- Institutional origin: Provincia de Buenos Aires educational administration
- Document or record class: provincial administrative records, platform state, official procedural entries, and related institutional data
- Intended governance use: declared provincial reference source for expected-event support, review context, and verification of official administrative traces
- Temporal relevance: relevant to the period during which the platform record or state is institutionally valid
- Known limitations: access scope, visibility, and interpretation may depend on permissions, process type, and platform design
- Boundary note: this source must be treated as bounded by the visible and institutionally recognized record classes actually available to authorized SAD use

### 4. COBOL or Legacy Provincial Data Source

- Source name: COBOL or legacy provincial data source
- Institutional origin: legacy provincial educational administration systems
- Document or record class: historical or legacy data records, reference outputs, and system-derived administrative traces
- Intended governance use: legacy evidence support, temporal comparison, and historical reconstruction where newer sources are incomplete
- Temporal relevance: especially relevant for historical periods, legacy continuity, and migration-sensitive review
- Known limitations: data structure, access conditions, provenance clarity, and completeness may be uneven
- Boundary note: legacy source use must remain explicitly bounded and should not be treated as unrestricted institutional truth outside its declared historical role

### 5. Google Sheets Operational Spreadsheets

- Source name: Google Sheets operational spreadsheets
- Institutional origin: SAD Moron operational workspaces and institutionally maintained spreadsheet practices
- Document or record class: operational tracking sheets, working registries, planning sheets, and administrative coordination artifacts
- Intended governance use: operational support evidence, cross-check context, and bounded review artifacts when formally recognized for a process
- Temporal relevance: usually tied to active operational periods and recent working cycles
- Known limitations: spreadsheet records may be mutable, derivative, collaborative, and unevenly formalized
- Boundary note: spreadsheets are not automatically authoritative; each use must remain bounded by declared process role and institutional recognition

### 6. Official Administrative Acts

- Source name: official administrative acts
- Institutional origin: competent educational or administrative authority
- Document or record class: resolutions, dispositions, decrees, acts, and formally issued administrative decisions
- Intended governance use: authoritative support for formal decisions, obligations, and institutional event confirmation
- Temporal relevance: tied to issuance date, period of validity, and any formal supersession
- Known limitations: formal acts may require contextual linkage to operational records in order to interpret process state fully
- Boundary note: only formally issued acts within declared scope should be treated as authoritative under this source class

### 7. Institutional Communications / Notifications

- Source name: institutional communications / notifications
- Institutional origin: SAD Moron, provincial administration, or other competent institutional actor
- Document or record class: notices, notifications, formal communications, and acknowledged institutional messages
- Intended governance use: evidence of notification-related expected events, communication traceability, and bounded review context
- Temporal relevance: tied to dispatch, receipt, acknowledgment, and applicable response periods
- Known limitations: communications may show notification state without fully proving downstream action completion
- Boundary note: informal messaging or communication outside declared institutional channels must not be treated as equivalent to formal notification by default

## Summary

This catalog provides the initial declared-source scaffold required for governance review.

It must be refined through governed change control before being treated as a final institutional catalog.
