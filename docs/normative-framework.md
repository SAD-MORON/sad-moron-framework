# Normative Framework

## Purpose

This document defines the initial normative framework for `SAD-MORON-FRAMEWORK`.

## Normative Intent

The repository establishes a bounded institutional governance layer for SAD Moron.

Normative meaning in this repository concerns:
- what may count as a declared source
- what may count as an expected administrative event
- how omission review is bounded
- how review outcomes should remain accountable and reconstructable

## Normative Objects

The following objects are intended to become normative for this repository:
- source definitions
- event definitions
- review scope definitions
- omission review rules
- audit decision formats

## Normative Rules

### Rule 1: Declared-source priority

No institutional claim should be treated as supported unless it references a declared source or explicitly states that support is absent.

### Rule 2: Expected-event clarity

No omission review should be initiated unless the expected administrative event is clearly defined.

### Rule 3: Period definition

No omission finding should be treated as valid unless the review period is defined.

### Rule 4: Bound review scope

Every review must define:
- the source set
- the event expectation
- the applicable period
- the applicable governance rule

### Rule 5: Human accountability

Final accountable decisions must remain attributable to authorized human review.

### Rule 6: Reconstruction readiness

Review records should preserve enough context for later institutional reconstruction.

### Rule 7: No hidden expansion

Operational growth must not silently redefine the normative core.

## Operational vs Normative Distinction

Normative documents define institutional meaning.

Operational documents define how that meaning is applied in review practice.

If an operational document appears to change institutional meaning, the normative layer takes precedence until the normative layer is explicitly updated.

## Current Normative Boundaries

This framework is currently bounded to:
- institutional documentation
- conceptual protocols
- audit-facing governance structure

It does not currently establish normative authority for software behavior, system execution, or external integrations.

## Governance Discipline

Changes to the normative framework should be:
- explicit
- documented
- reviewable
- compatible with declared SAD Moron scope

## Summary

The normative framework is intentionally small.

Its function is to stabilize meaning before any future operational or technical expansion.
